Erfahrungen mit Okta sichern

In diesem Thema erfahren Sie, wie Sie Erfahrungen mit Okta SSO sichern.

Wenn ein Erlebnis mit SSO gesichert ist, müssen sich Benutzer authentifizieren, bevor sie auf ein Web- oder Mobile App-Erlebnis zugreifen können.

Hinweis: Die Schritte und Bilder in diesem Thema verwenden die Testversion von Okta SSO.

Bei den Schritten in diesem Thema wurde davon ausgegangen, dass Sie sich für ein Okta-Testkonto angemeldet haben, sich in Ihrem Konto angemeldet haben und auf die Okta-Dashboard-Seite zugreifen können.

Erstellen einer Anwendung

Beginnen Sie mit dem Erstellen einer Anwendung.

  1. Melden Sie sich bei Ihrem Okta-Testkonto an.
  2. Klicken Anwendungen in der oberen Kopfzeile.
  3. Klicken Sie auf Anwendung hinzufügen.
  4. Auf der Anwendung hinzufügen Seite, klicken Sie Neue App erstellen.
  5. Wählen Sie a Plattform von Netz und ein Anmeldemethode von SAML 2.0.
  6. Klicken Schaffen.
  7. Geben Sie ein App Name.
  8. (Optional) Browse and select and App logo.
  9. Klicken Sie auf Weiter.
  10. Stellen Sie die Single Sign-On-URL zu https://auth.bcvp0rtal.com/login/callback.
  11. Für die Zielgruppen-URI (SP-Entitäts-ID) Geben Sie die Okta-URL für Ihre Okta-Startseite ein. Dies wird in der E-Mail des neuen Kontos aufgeführt, die Sie von Okta erhalten haben, als Ihr Testkonto erstellt wurde.
  12. In dem ATTRIBUTE ERKLÄRUNGEN Abschnitt sollten die folgenden Attribute erstellt werden:
    Hinweis: Die Attribute MÜSSEN genau wie hier gezeigt benannt werden.
    Nennen Wert
    Vorname user.firstName
    LastName user.lastName
    E-Mail senden Benutzer Email

    Die Zuordnung der Anwendungsattribute führt dazu, dass in Brightcove Engage Experiences Folgendes passiert:

    • Für mobile App-Erlebnisse werden der Name und die E-Mail-Adresse des Benutzers auf der Profilseite angezeigt
    • Für alle Erlebnisse wird die E-Mail-Adresse des Nutzers als Teil der Videoanalyse angezeigt
  13. Drücke den <> Vorschau der SAML-Zusicherung Taste. Gültiges XML sollte in einem neuen Browser-Tab angezeigt werden. Bei Fehlern wird eine Meldung angezeigt.
  14. Klicken Sie auf Weiter.
  15. Wählen Sie eine Option für Sind Sie Kunde oder Partner? und dann klicken Fertig.
  16. Bestätigen Sie, dass die Anwendung erstellt wurde und die die Einstellungen Seite erscheint.
  17. Suchen Sie, wo SAML 2.0 erst konfiguriert ist, wenn Sie die Setup-Anweisungen abgeschlossen haben. Klicken Setup-Anweisungen anzeigen.

Die So konfigurieren Sie SAML 2.0 Anweisungen werden in einem neuen Browser-Tab geöffnet. Diese Informationen werden beim Erstellen eines Zugriffssteuerungsprofils in Brightcove Engage benötigt.

Erstellen eines Zugriffssteuerungsprofils

Sie müssen ein Zugriffssteuerungsprofil erstellen, das verwendet wird, um alle Erfahrungen zu sichern, die Sie mit SSO sichern möchten.

  1. Melden Sie sich bei Ihrem Brightcove Engage-Konto an.
  2. Klicken Einstellungen in der Kopfzeile.
  3. Klicken Sie auf Profil erstellen.
  4. Gib dem Profil einen Namen.
  5. Kehren Sie zu den Anweisungen So konfigurieren Sie SAML 2.0 zurück. Kopiere das Single Sign-On-URL des Identitätsanbieters und füge es in die SAML 2.0-Endpunkt (HTTP) Gebiet.
  6. Kehren Sie zu den Anweisungen So konfigurieren Sie SAML 2.0 zurück. Kopieren Sie den gesamten Wert in die X.509-Zertifikat Feld und fügen Sie es in die X.509-Zertifikat Feld. Stellen Sie sicher, dass Sie die einschließen BEGIN / END CERTIFICATE Linien.
  7. Klicken Sie auf Speichern und bestätigen Sie, dass das Profil erstellt wurde.

Gruppenberechtigungen zuweisen

Der letzte Schritt in Okta besteht darin, allen Benutzern in Okta Zugriff auf die Brightcove Engage-Anwendung zu gewähren.

  1. Kehren Sie zum Okta-Dashboard zurück.
  2. Klicken Sie für die von Ihnen erstellte Okta-Anwendung auf Zuordnungen Tab.
  3. Klicken Zuordnen > Gruppen zuordnen.
  4. Klicken Zuordnen neben dem Jeder gruppieren und dann klicken Getan.

Weisen Sie das Zugriffssteuerungsprofil einem Erlebnis zu

Der letzte Schritt besteht darin, das Zugriffssteuerungsprofil zuzuweisen, das erstellt wurde, den Erlebnissen, die Sie sichern möchten.

  1. Zurück zur Seite Brightcove Engage.
  2. Bearbeiten Sie das Erlebnis, das Sie sichern möchten.
  3. Klicken Sie in der linken Navigation auf ZUGANGSKONTROLLE.
  4. Wähle aus Zugriffssteuerungsprofil Dies ist mit der Okta-SSO-Konfiguration verbunden.
  5. Klicken Sie auf Speichern.
  6. Veröffentlichen Sie das Erlebnis.

Wenn Sie auf die Erfahrung zugreifen, sollten Sie von Okta aufgefordert werden, sich anzumelden. Das folgende Bild zeigt, wenn versucht wird, auf eine Mobile App Experience zuzugreifen.
