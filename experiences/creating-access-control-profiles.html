Erstellen von Zugriffssteuerungsprofilen

In diesem Thema erfahren Sie, wie Sie Zugriffssteuerungsprofile erstellen.

Wenn ein Brightcove Engage™ Erfahrung wird veröffentlicht, die Erfahrungen sind öffentlich:

  • Mobile Apps - Jeder mit dem Zugangsschlüssel und kann auf die App zugreifen
  • Web Applikationen - Jeder mit der Site-URL kann auf das Webportal zugreifen

Zugriffssteuerungsprofile können erstellt werden, sodass Benutzer gezwungen sind, sich mit gültigen Anmeldeinformationen zu authentifizieren, bevor sie auf die App / Site zugreifen können. Brightcove Engage unterstützt den SAML 2.0-Standard (Security Assertion Markup Language 2.0) für den Austausch von Authentifizierungs- und Autorisierungsdaten zwischen Sicherheitsdomänen.

Brightcove Engage unterstützt die meisten wichtigen Identitätsanbieter, darunter:

  • Klingeln
  • OneLogin
  • Auth0
  • ADFS 2.0/3.0
  • Google G Suite
  • Shibboleth
  • Okta
  • Salesforce.com

Andere Anbieter, die den SAML 2.0-Standard verwenden, werden ebenfalls arbeiten.

Hinweis: Für Brightcove Engage ist SAML 2.0 für die Single Sign-On-Authentifizierung erforderlich. SAML 1.1 wird nicht unterstützt.

Ordnen Sie bei der Konfiguration Ihrer SSO-Software die folgenden Attribute zu:

  • Vorname wird Vorname zugeordnet
  • lastName wird dem Nachnamen zugeordnet
  • E-Mail Karten zu E-Mail

Neues Profil erstellen

Zugriffssteuerungsprofile werden als Teil der Projekteinstellungen erstellt und dann den Erfahrungen zugewiesen.

Gehen Sie folgendermaßen vor, um ein Zugriffssteuerungsprofil zu erstellen:

  1. Melden Sie sich bei Brightcove Engage an.
  2. Von dem Projekte Seite, klicken Sie die Einstellungen.
  3. Klicken Sie in der linken Navigation auf Zugriffssteuerungsprofile. Die Seite Zugriffssteuerungsprofile wird mit einer Liste der Profile (falls vorhanden) angezeigt, die erstellt wurden.
  4. Klicken Sie auf Profil erstellen.
  5. Gib dem Profil einen Namen.
  6. (Optional) Configure a Session Timeout. Dies ist die Zeitspanne, in der ein Benutzer bei einer App angemeldet sein kann, bevor er sich erneut authentifizieren muss. Ein paar Anmerkungen:
    • Ihr SSO-Anbieter muss so eingestellt sein, dass er entweder eine erneute Authentifizierung erzwingt oder ein Sitzungszeitlimit von weniger als diesem Zeitlimit aufweist. Wenn dies nicht der Fall ist, werden wir nach Ablauf unserer Zeitüberschreitung zu Ihrem SSO-Anbieter weiterleiten und dieser gibt sofort eine Antwort aus, dass der Benutzer angemeldet ist.
  7. Wähle aus SSO Möglichkeit. Die SSO-Konfigurationsoptionen werden angezeigt.
  8. Wenn Sie eine Exportdatei Ihrer SAML 2.0-Metadaten haben, klicken Sie auf Datei aussuchen Durchsuchen Sie dann die Metadatendatei und wählen Sie sie aus. Dies wird die SAML 2.0-Endpunkt und der X.509-Zertifikat Felder.
  9. Geben Sie einen Wert für den SAML 2.0 Endpoint (HTTP) ein. Dieser Wert sollte bei Ihrem SAML-Anbieter verfügbar sein.
  10. Geben Sie einen Wert für die ein X.509-Zertifikat. Dieser Wert sollte bei Ihrem SAML-Anbieter verfügbar sein.
  11. (Optional) Enter the SAML Issuer / Entity ID Override. Geben Sie hier einen Wert ein, um den SAML-Aussteller optional zu überschreiben.
  12. (Optional) Enter the ACS URL Override. Geben Sie hier einen Wert ein, um optional die ACS-URL zu überschreiben.
  13. (Optional) Enter the SAML AuthnContext Override. Geben Sie hier einen Wert ein, um optional den SAML AuthnContext zu überschreiben.
  14. (Optional) Enter any SAML attributes for that you want passed to your external JavaScript.
  15. (Optional) Check Expose all attributes to expose all of the SAML attributes for use in external JavaScript.
  16. Klicken Sie auf Speichern.
  17. Überprüfen Sie, ob das von Ihnen erstellte Profil in der Liste der Zugriffssteuerungsprofile angezeigt wird.

Um ein Zugriffskontrollprofil zu bearbeiten oder zu löschen, klicken Sie auf das Bearbeiten () or delete () icon associated with the profile. Sobald Zugriffssteuerungsprofile erstellt wurden, können sie einer Site als Teil der Site-Eigenschaften zugewiesen werden. Weitere Informationen finden Sie unter Kontrollieren des Zugriffs auf eine Erfahrung.

Konfigurieren der ACS-URL

Ihr SSO-Anbieter benötigt eine ACS-URL (Assertion Consumer Service URL). Die ACS-URL wird im Abschnitt SSO der Tabelle Zugriffssteuerungsprofile aufgeführt.

Die ACS-URL lautet standardmäßig https://auth.bcvp0rtal.com/login/callback. Sie können die ACS-URL auch überschreiben, indem Sie Ihre eigene URL angeben. Beide Optionen können zum Konfigurieren strengerer SSO-Anbieter verwendet werden.

Hinweis: Bei Verwendung eines IDP-initiierten SSO-Workflows sollte die in Ihrer SSO-Software angegebene ACS-URL auf eine Brightcove Engage-Site verweisen, die mit diesem Zugriffssteuerungsprofil konfiguriert ist. In diesem Fall sollte die https://auth.bcvp0rtal.com/login/callback URL nicht verwendet werden.
