Vorschau und Veröffentlichung eines Weberlebnisses

In diesem Thema erfahren Sie, wie Sie eine Vorschau einer Web Experience anzeigen und veröffentlichen.

Web-Erlebnisse können während der Entwicklung problemlos in der Vorschau angezeigt werden. Links zur Vorschau der Erfahrung können ebenfalls geteilt werden. Wenn die Erfahrung abgeschlossen ist, kann sie veröffentlicht werden, um sie öffentlich zugänglich zu machen.

Vorschau einer Web-Erfahrung

In der Vorschau einer Web-Erfahrung können Sie sehen, wie die Erfahrung auf Mobil-, Tablet- und Desktop-Geräten angezeigt wird. Um eine Vorschau einer Web-Erfahrung anzuzeigen, bearbeiten Sie die Erfahrung und klicken Sie dann auf Vorschau im Experience Editor.

Es gibt zwei Möglichkeiten:

  • Responsive Preview - Öffnet das Erlebnis in einem neuen Browserfenster zur Vorschau
  • Shareable Link abrufen - Bietet eine Vorschau-URL, damit die Vorschau mit anderen geteilt werden kann.

Die generierte gemeinsam nutzbare URL ist 72 Stunden gültig.

Hinweis: Wenn Sie ein Sitzungstimeout im Rahmen von Einstellungen für die Zugriffssteuerung konfiguriert haben, überschreibt das 72-Stunden-Timeout das konfigurierte Sitzungstimeout. Wenn das Erlebnis veröffentlicht wird, wird das Sitzungstimeout (falls vorhanden) durchgesetzt.

Veröffentlichen einer Web-Erfahrung

Durch das Veröffentlichen einer Web Experience wird sie den Zuschauern zur Verfügung gestellt und eine statische URL für die Site generiert. Gehen Sie folgendermaßen vor, um eine Erfahrung zu veröffentlichen:

Hinweis: Wenn Sie kürzlich Änderungen an Video-Metadaten oder Wiedergabelisten vorgenommen haben, kann es einige Minuten dauern, bis diese Änderungen bei veröffentlichten Erlebnissen angezeigt werden.
  1. Klicken Sie auf den Erfahrungsnamen, um die Site im Erfahrungseditor zu öffnen.
  2. Klicken Sie auf Veröffentlichen.

Veröffentlichen von Änderungen an einer Web Experience

Wenn Sie Änderungen an einem Erlebnis vornehmen, muss es erneut veröffentlicht werden. Erfahrungen, die erneut veröffentlicht werden müssen, zeigen den Erlebnisstatus von unveröffentlichten Änderungen an.

Befolgen Sie die Schritte unter, um die Änderungen erneut zu veröffentlichen Veröffentlichen einer Web-Erfahrung Abschnitt dieses Themas.

Hinweis: Möglicherweise müssen Sie den Browser-Cache und die Cookies löschen, wenn keine Änderungen angezeigt werden.

Web Experience offline schalten

Wenn Sie eine Erfahrung offline schalten, ist sie unter ihrer URL nicht mehr verfügbar. Um ein Erlebnis offline zu schalten, klicken Sie auf eine Zeile und dann auf Offline nehmen. Um zu bestätigen, dass das Erlebnis offline geschaltet werden soll, klicken Sie auf Offline nehmen. Um die Erfahrung wieder verfügbar zu machen, muss die Erfahrung erneut veröffentlicht werden. Sobald die Erfahrung offline ist, ist sie nicht mehr über ihre URL erreichbar.

Hinweis: Wenn Sie die Erfahrung offline schalten, wird das Erlebnis nicht gelöscht.
