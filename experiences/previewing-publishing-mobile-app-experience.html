Vorschau und Veröffentlichung einer mobilen App-Erfahrung

In diesem Thema erfahren Sie, wie Sie eine Vorschau und Veröffentlichung einer Mobile App Experience anzeigen.

In der Vorschau einer mobilen App-Erfahrung können Sie sehen, wie die Erfahrung auf einem mobilen Gerät angezeigt wird. Um eine Vorschau einer Mobile App-Erfahrung anzuzeigen, bearbeiten Sie die Erfahrung und klicken Sie dann auf Vorschau im Experience Editor.

Es gibt zwei Möglichkeiten:

  • Responsive Preview - Öffnet das Erlebnis in einem neuen Browserfenster zur Vorschau
  • Shareable Link abrufen - Bietet eine Vorschau-URL, damit die Vorschau mit anderen geteilt werden kann.

Die generierte gemeinsam nutzbare URL ist 72 Stunden gültig.

Hinweis: Wenn Sie ein Sitzungstimeout im Rahmen von Einstellungen für die Zugriffssteuerung konfiguriert haben, überschreibt das 72-Stunden-Timeout das konfigurierte Sitzungstimeout. Wenn das Erlebnis veröffentlicht wird, wird das Sitzungstimeout (falls vorhanden) durchgesetzt.

Veröffentlichen einer Mobile App Experience

Durch das Veröffentlichen einer Mobile App Experience wird ein Zugriffsschlüssel generiert. Der Zugriffsschlüssel wird von Brightcove Engage benötigt™ App (iOS und Android), um auf das Erlebnis zuzugreifen. Informationen zum Installieren und Verwenden der Brightcove Engage-App finden Sie unter Verwenden der Brightcove Engage-Anwendung.

Gehen Sie folgendermaßen vor, um eine Mobile App Experience zu veröffentlichen:

Hinweis: Wenn Sie kürzlich Änderungen an Video-Metadaten oder Wiedergabelisten vorgenommen haben, kann es einige Minuten dauern, bis diese Änderungen bei veröffentlichten Erlebnissen angezeigt werden.
  1. Klicken Sie auf den Erfahrungsnamen, um ihn im Erfahrungseditor zu öffnen.
  2. Klicken Sie auf Veröffentlichen.
  3. Klicken Sie im Bestätigungsdialogfeld für die Veröffentlichung auf Jetzt veröffentlichen.
  4. Die Erfahrung wird veröffentlicht und der Zugangsschlüssel wird angezeigt.

    Sobald die Erfahrung veröffentlicht wurde, wird der Zugriffsschlüssel als Teil der Erfahrungsdetails auf der Erfahrungshomepage angezeigt.

Verwenden der Brightcove Engage-Anwendung

Nachdem eine mobile App-Erfahrung veröffentlicht wurde, wird der Zugriffsschlüssel in die mobile Brightcove Engage-App eingegeben, um die Erfahrung anzuzeigen. Informationen zum Installieren und Verwenden der Brightcove Engage-Anwendung finden Sie unter Verwenden der Brightcove Engage-Anwendung.

Veröffentlichen von Änderungen an einer Erfahrung

Wenn Sie Änderungen an einem Erlebnis vornehmen, muss es erneut veröffentlicht werden. Erfahrungen, die erneut veröffentlicht werden müssen, zeigen den Erlebnisstatus von unveröffentlichten Änderungen an.

Befolgen Sie die Schritte unter, um die Änderungen erneut zu veröffentlichen Veröffentlichen einer Mobile App Experience Abschnitt dieses Themas.

Mobile App-Erfahrung offline schalten

Wenn eine Mobile App Experience offline geschaltet wird, kann von nativen mobilen Anwendungen nicht mehr auf sie zugegriffen werden. Um eine Mobile App Experience offline zu schalten, klicken Sie auf eine Mobile App Experience und dann auf Offline nehmen. Um zu bestätigen, dass das Erlebnis offline geschaltet werden soll, klicken Sie auf Offline nehmen. Um die Erfahrung wieder verfügbar zu machen, muss die Erfahrung erneut veröffentlicht werden.

Hinweis: Wenn Sie die Erfahrung offline schalten, wird das Erlebnis nicht gelöscht.
