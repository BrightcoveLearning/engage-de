Zuweisen einer benutzerdefinierten Domäne zu einer Weberfahrung

In diesem Thema erfahren Sie, wie Sie einer Web-Experience eine benutzerdefinierte Domäne zuweisen.

Das Zuweisen einer benutzerdefinierten Domäne zu einer Web Experience bietet die Möglichkeit, über eine leicht zu merkende URL auf die Website zuzugreifen (im Gegensatz zu einer site-slug.bcvp0rtal.com URL). Wenn Sie benutzerdefinierte Domänen verwenden, können Sie der URL auch einen optionalen Pfad zuweisen. Mithilfe von Pfaden können Sie verschiedene Vorlagen verwenden, Microsites, Kampagnen, interne Portale usw. erstellen, während Sie immer noch dieselbe Domäne verwenden. Zum Beispiel videos.mycustomdomain.com kann erweitert werden, um auch Pfade wie videos.mycustomdomain.com/event oder videos.mycustomdomain.com/product. Pfade ermöglichen es Publishern, die mehrere Websites und SSL haben, nur SSL für eine Domäne und nicht für jeden einzelnen Site-Slug einrichten zu müssen.

Hinweis: Benutzerdefinierte Domänen müssen im Rahmen der Projekteinstellungen erstellt werden, bevor sie einem Erlebnis zugewiesen werden können. Weitere Informationen finden Sie unter Konfigurieren von benutzerdefinierten Domänen und SSL.

Gehen Sie folgendermaßen vor, um einer Web-Experience eine benutzerdefinierte Domäne zuzuweisen:

  1. Bearbeiten Sie das Erlebnis.
  2. Klicken Sie im linken Navigationsbereich auf SITE-KONFIGURATION > URL. Die der Site zugewiesene erste Domäne wird angezeigt.
  3. (Optional) Deaktivieren Sie „ Aktiv“? Kontrollkästchen, um das zu deaktivieren site-xxxx.bcvp0rtal.com domain. In der Regel deaktivieren Publisher das site-xxxx.bcvp0rtal.com, wenn sie eine benutzerdefinierte Domäne verwenden, sodass es nicht von Suchmaschinen indiziert wird.
  4. Klicken Sie auf Benutzerdefinierte Domäne hinzufügen.
  5. Standardmäßig sind Domänen Aktiv. Durch das Deaktivieren von Aktiv wird Brightcove Engage vom Routing für den Eintrag deaktiviert, aber die Kombination aus Pfad/Domäne für die Verwendung auf einer anderen Site wird nicht freigegeben.
  6. Wählen Sie eine Domäne aus der Dropdown-Liste aus. Die Liste der benutzerdefinierten Domänen, die im Rahmen der Projekteinstellungen konfiguriert wurden, wird angezeigt.
  7. Geben Sie optional einen Pfad für die Domäne ein.
  8. Fügen Sie zusätzliche Domänen für die Site hinzu, indem Sie auf Benutzerdefinierte Domäne hinzufügen klicken.
  9. Wenn Sie fertig sind, klicken Sie auf Speichern.

Klicken Sie auf das Löschsymbol () um eine benutzerdefinierte Domäne zu entfernen.

Hinweis: Damit eine der Domainänderungen (einschließlich des Aktivierens oder Löschens einer Domain) wirksam wird, muss die Website veröffentlicht werden.

Die Liste der benutzerdefinierten Domänen wird im Veröffentlichungsdialog aufgeführt.

Um eine der benutzerdefinierten Domänen zu deaktivieren oder zu entfernen, bearbeiten Sie die Site, klicken Sie auf SITE-KONFIGURATION > URL und deaktivieren Sie das Kontrollkästchen Aktiv, um die Domäne zu deaktivieren, oder klicken Sie auf das Löschsymbol () um die benutzerdefinierte Domäne zu entfernen.
