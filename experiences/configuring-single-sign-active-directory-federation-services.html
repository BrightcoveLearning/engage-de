Konfigurieren von Single Sign-On mit Active Directory-Verbunddiensten

In diesem Thema erfahren Sie, wie Sie Brightcove Engage so konfigurieren,™ dass es Single Sign-On mit Active Directory-Verbunddiensten (ADFS) unterstützt.

Bevor du anfängst

Führen Sie diese Schritte zur Ersteinrichtung ab:

  1. Aktivieren Sie RelayState für die IDP-initiierte Anmeldung, indem Sie den Anweisungen hier folgen. Der Speicherort variiert je nach verwendeter ADFS-Version.
  2. Starten Sie den ADFS-Dienst neu, damit die Änderungen übernommen werden.

Hinzufügen eines Zugriffssteuerungsprofils

  1. Öffnen Sie Verwaltung im Startmenü oder in der Systemsteuerung von Windows und öffnen Sie dann die AD FS-Verwaltungsanwendung.
  2. Öffnen Sie Service > Zertifikate im linken Explorer-Panel.
  3. Doppelklicken Sie auf das Token-Signaturzertifikat, das Sie verwenden möchten.
  4. Klicken Sie auf die Registerkarte Details und dann auf In Datei kopieren...
  5. Klicken Sie im Exportassistenten auf Weiter und wählen Sie dann die Base-64-codierte X.509 (.CER) Option aus.
  6. Speichern Sie die Zertifikatsdatei in Ihrem lokalen Dateisystem. Öffnen Sie das Zertifikat in einem Texteditor. Dies ist das Zertifikat, das zum Einrichten des Zugriffskontrollprofils in Brightcove Engage verwendet wird.
  7. Öffne Brightcove Engage.
  8. Klicken Sie auf Projekte > Einstellungen.
  9. Klicken Sie im linken Navigationsbereich auf Zugriffssteuerungsprofile.
  10. Klicken Sie auf Profil erstellen.
  11. Geben Sie ihm einen Namen und wählen Sie dann SSO - Benötigt einen Benutzernamen und ein Passwort für den Zugriff.
  12. Geben Sie im Feld SAML 2.0 Endpoint (HTTP) die URL zur IdP-initiierten Anmeldeseite für Ihren ADFS-Server ein. Dies endet normalerweise damit IdpInitiatedSignOn.aspx. Wenn beispielsweise der SP-initiierte Anmeldelink lautet https://sso-test.brightcove.com/adfs/ls, lautet die IdP-initiierte Anmeldeseite https://sso-test.brightcove.com/adfs/ls/IdpInitiatedSignOn.aspx.
  13. Aktivieren Sie die Option Mein SSO System ist ADFS (Active Directory-Verbunddienste).
  14. Kopieren Sie das in Schritt 6 gespeicherte Zertifikat und fügen Sie es in das Feld X.509-Zertifikat ein.
  15. Klicken Sie auf Speichern, um das Zugriffssteuerungsprofil zu speichern. Das neue Zugriffskontrollprofil sollte in der Profilliste erscheinen.
  16. Öffnen Sie in der AD FS-Management-Anwendung Relying Party Trusts.
  17. Klicken Sie im rechten Bereich auf Vertrauensvertrauen hinzufügen....
  18. Klicken Sie im Assistenten auf Start und fügen Sie dann die Metadaten-URL, die in Schritt 15 angezeigt wurde, in das Adressfeld Federation Metadaten ein.
  19. Klicken Sie auf Weiter und geben Sie dann Ihrem Vertrauensvertrauen einen Namen.
  20. Klicken Sie weiter auf Weiter bis zum Schritt Fertig stellen. Deaktivieren Sie das Kontrollkästchen Richtlinie zur Anspruchsausstellung für diese Anwendung konfigurieren. Klicken Sie auf Schließen.

Sie sollten nun in der Lage sein, dem erstellten Zugriffssteuerungsprofil eine Erfahrung zuzuordnen. Nachdem die Erfahrung veröffentlicht wurde, sollte die Navigation zu ihr den ADFS-Anmeldeworkflow durchlaufen.
