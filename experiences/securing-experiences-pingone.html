Sichern von Erfahrungen mit PingOne

In diesem Thema erfahren Sie, wie Sie Erfahrungen mit PingOne SSO sichern.

Wenn ein Erlebnis mit SSO gesichert ist, müssen sich Benutzer authentifizieren, bevor sie auf ein Web- oder Mobile App-Erlebnis zugreifen können.

Hinweis: Die Schritte und Bilder in diesem Thema verwenden die Testversion von PingOne SSO.

Bei den Schritten in diesem Thema wurde davon ausgegangen, dass Sie ein PingOne-Testkonto erstellt und Ihr Konto per E-Mail bestätigt haben.

Erstellen eines Identitätsrepositorys

Beginnen Sie mit der Erstellung eines Identitätsrepositorys.

  1. Melden Sie sich bei Ihrem PingOne-Konto an. Das Instrumententafel Seite wird angezeigt.
  2. Klicken KONFIGURATION in der Kopfzeile. Das die Einstellungen Seite wird geöffnet.
  3. Klicken Stellen Sie eine Verbindung zu einem Identitäts-Repository her.
  4. Klicke auf PINGONVERZEICHNIS und dann klicken Nächster.
  5. Bestätigen Sie, dass das Verzeichnis konfiguriert wurde.
  6. Klicken Sie auf Weiter.
  7. Für die KARTENATTRIBUTE Behalten Sie die Standardwerte bei und klicken Sie auf Fertig.
Auf der Seite PingOne-Verzeichnis werden die Verzeichnisstatistiken angezeigt.

Erstellen einer Anwendung

Als nächstes erstellen wir eine Anwendung, die mit SSO gesichert wird.

  1. Klicken ANWENDUNGEN in der Kopfzeile. Das meine Anwendungen Seite wird geöffnet.
  2. Klicken Sie auf Anwendung hinzufügen.
  3. Wählen Neue SAML-Anwendung.
  4. Geben Sie ein Anwendungsname und Anwendungsbeschreibung.
  5. Wählen Sie a Kategorie (empfehlen Kommunikation).
  6. (Optional) Click Change and browse and select an icon for the application.
  7. Klicken Sie auf Weiter mit dem nächsten Schritt.
  8. Auf der Anwendungskonfiguration Seite, suchen Sie die SAML-Metadaten Feld.
  9. Drücke den Herunterladen Verknüpfung.
Die heruntergeladene Datei wird beim Erstellen eines Zugriffssteuerungsprofils in Brightcove Engage benötigt.

Erstellen eines Zugriffssteuerungsprofils

Sie müssen ein Zugriffssteuerungsprofil erstellen, das verwendet wird, um alle Erfahrungen zu sichern, die Sie mit SSO sichern möchten.

  1. Melden Sie sich bei Ihrem Brightcove Engage-Konto an.
  2. Klicken Einstellungen in der Kopfzeile.
  3. Klicken Sie in der linken Navigation auf Zugriffssteuerungsprofile.
  4. Klicken Sie auf Profil erstellen.
  5. Gib dem Profil einen Namen.
  6. Suchen Sie die Importieren Sie SSO-Metadaten Abschnitt und dann klicken Datei aussuchen.
  7. Durchsuchen Sie die SAML-Metadatendatei, die Sie von PingOne heruntergeladen haben, und wählen Sie sie aus (im vorherigen Abschnitt).
  8. Vergewissern Sie sich, dass Werte für den SAML 2.0 Endpoint und das X.509-Zertifikat ausgefüllt wurden.
  9. Klicken Sie auf Speichern und bestätigen Sie, dass das Profil erstellt wurde.
  10. Klick auf das Metadaten-URL Feld und speichern Sie die XML-Datei.

Die XML-Datei wird benötigt, um die Anwendungskonfiguration in PingOne abzuschließen.

Abschluss der Anwendungskonfiguration

  1. Kehren Sie zur PingOne-Anwendung zurück.
  2. Auf der Anwendungskonfiguration Abschnitt finden Sie die Metadaten hochladen Feld.
  3. Klicken Datei aussuchen Durchsuchen Sie die Datei, die Sie nach dem Erstellen des Zugriffssteuerungsprofils in Brightcove Engage heruntergeladen haben, und wählen Sie sie aus.
  4. Bestätige das Assertion Consumer Service (ACS) und Entitäts-ID Felder wurden nach dem Import mit Werten ausgefüllt.
  5. Klicken Sie auf Weiter mit dem nächsten Schritt.
  6. Auf der SSO-Attributzuordnung Abschnitt, fügen Sie die folgenden Attribute hinzu:
    Hinweis: Die Attribute MÜSSEN genau wie hier gezeigt benannt werden.
    Anwendungsattribut Literaler Wert
    Vorname Vorname
    LastName Nachname
    E-Mail senden Email
  7. Klicken Sie auf Weiter mit dem nächsten Schritt.
  8. Auf der Gruppenzugriff Seite, suchen Sie die Benutzer @ Verzeichnis gruppieren und dann klicken Hinzufügen.
  9. Klicken Sie auf Weiter mit dem nächsten Schritt.
  10. Klicken Sie auf Beenden.
  11. Bestätigen Sie, dass die Anwendung ist Aktiv und aktiviert.

Die Zuordnung der Anwendungsattribute führt dazu, dass in Brightcove Engage Experiences Folgendes passiert:

  • Für mobile App-Erlebnisse werden der Name und die E-Mail-Adresse des Benutzers auf der Profilseite angezeigt
  • Für alle Erlebnisse wird die E-Mail-Adresse des Nutzers als Teil der Videoanalyse angezeigt

Weisen Sie das Zugriffssteuerungsprofil einem Erlebnis zu

Der letzte Schritt besteht darin, das Zugriffssteuerungsprofil zuzuweisen, das erstellt wurde, den Erlebnissen, die Sie sichern möchten.

  1. Zurück zur Seite Brightcove Engage.
  2. Bearbeiten Sie das Erlebnis, das Sie sichern möchten.
  3. Klicken Sie in der linken Navigation auf SITE-KONFIGURATION > Zugriffssteuerung.
  4. Wähle aus Zugriffssteuerungsprofil Dies ist mit der PingOne-SSO-Konfiguration verbunden.
  5. Klicken Sie auf Speichern.
  6. Veröffentlichen Sie das Erlebnis.

Wenn Sie auf die Erfahrung zugreifen, sollten Sie von PingOne aufgefordert werden, sich anzumelden. Das folgende Bild zeigt, wenn versucht wird, auf eine Mobile App Experience zuzugreifen.
