    Sichern von Erlebnissen mit PingOne

    In diesem Thema erfahren Sie, wie Sie Erlebnisse mit PingOne SSO sichern können.

    Wenn ein Erlebnis mit SSO gesichert wird, müssen sich Benutzer authentifizieren, bevor sie auf eine Web- oder Mobile-App-Erfahrung zugreifen können.

    Hinweis: Die Schritte und Bilder in diesem Thema verwenden die Testversion von PingOne SSO.

    Die Schritte in diesem Thema gehen davon aus, dass Sie ein PingOne-Testkonto erstellt und Ihr Konto per E-Mail bestätigt haben.

    Erstellen eines Identitäts-Re

    Beginnen Sie mit der Erstellung eines Identitäts-Re

    1. Melden Sie sich bei Ihrem PingOne-Konto an. Die Dashboard-Seite wird angezeigt.
    2. Klicken Sie im Header auf SETUP . Die Seite „Einstellungen “ wird geöffnet.
    3. Klicken Sie auf Mit einem Identity Repository verbinden.
    4. Klicken Sie auf PINGONE DIRECTORY und dann auf Next.
    5. Bestätigen Sie, dass das Verzeichnis konfiguriert wurde.
    6. Klicken Sie auf Next (Weiter).
    7. Behalten Sie im Abschnitt MAP ATTRIBUTES die Standardwerte bei und klicken Sie auf Fertig.
    Auf der Seite PingOne Directory werden die Verzeichnisstatistiken angezeigt.

    Erstellen einer Anwendung

    Als Nächstes erstellen wir eine Anwendung, die mit SSO gesichert wird.

    1. Klicken Sie im Header auf APPLICATION Die Seite „Meine Anwendungen “ wird geöffnet.
    2. Klicken Sie auf „ Anwendung hinzufügen“
    3. Wählen Sie Neue SAML-Anwendungaus.
    4. Geben Sie einen Anwendungsnamen und eine Anwendungsbeschreibungein.
    5. Wählen Sie eine Kategorie aus (empfehlen Sie Kommunikation).
    6. ( Optional) Klicken Sie auf Ändern , durchsuchen Sie und wählen Sie ein Symbol für die Anwendung aus.
    7. Klicken Sie auf Weiter zum nächsten Schritt.
    8. Suchen Sie auf der Seite Application Configuration das Feld SAML-Metadaten .
    9. Klicken Sie auf den Link Download .
    Die heruntergeladene Datei wird benötigt, wenn ein Zugriffssteuerungsprofil in Brightcove Engage erstellt wird.

    Erstellen eines Zugriffssteuerungsprofils

    Sie müssen ein Zugriffssteuerungsprofil erstellen, das verwendet wird, um alle Erlebnisse zu sichern, die Sie mit SSO sichern möchten.

    1. Melden Sie sich bei Ihrem Brightcove Engage Konto an.
    2. Klicken Sie in der Kopfzeile auf Einstellungen .
    3. Klicken Sie in der linken Navigationsleiste auf Profile der Zugriffssteuerung.
    4. Klicken Sie auf Create Profile.
    5. Geben Sie dem Profil unter Name einen Namen.
    6. Suchen Sie den Abschnitt SSO-Metadaten importieren und klicken Sie dann auf Datei auswählen.
    7. Durchsuchen und wählen Sie die SAML-Metadatendatei aus, die Sie von PingOne heruntergeladen haben (im vorherigen Abschnitt).
    8. Stellen Sie sicher, dass die Werte für den SAML 2.0 Endpoint und das X.509-Zertifikat ausgefüllt wurden.
    9. Klicken Sie auf „ Speichern “ und bestätigen Sie, dass das Profil erstellt wurde.
    10. Klicken Sie auf das Feld Metadaten-URL und speichern Sie die XML-Datei.

    Die XML-Datei wird benötigt, um die Anwendungskonfiguration in PingOne abzuschließen.

    Vervollständigen der Anwendungskonfiguration

    1. Kehren Sie zur PingOne-Anwendung zurück.
    2. Suchen Sie im Abschnitt Anwendungskonfiguration das Feld Updates-Metadaten .
    3. Klicken Sie auf Datei auswählen und durchsuchen und wählen Sie die Datei aus, die Sie nach dem Erstellen des Access Control-Profils in Brightcove Engage heruntergeladen haben.
    4. Bestätigen Sie, dass die Felder Assertion Consumer Service (ACS) und Entity ID nach dem Import mit Werten ausgefüllt wurden.
    5. Klicken Sie auf Weiter zum nächsten Schritt.
    6. Fügen Sie im Abschnitt SSO-Attributzuordnung die folgenden Attribute hinzu:
      Hinweis: Die Attribute MÜSSEN genau wie hier gezeigt benannt werden.
      Anwendungs-Attri Wörtlicher Wert
      firstName Vorname
      Nachname-Name Nachname
      E-Mail E-Mail
    7. Klicken Sie auf Weiter zum nächsten Schritt.
    8. Suchen Sie auf der Gruppenzugriffsseite die Gruppe Users @directory und klicken Sie dann auf Add.
    9. Klicken Sie auf Weiter zum nächsten Schritt.
    10. Klicken Sie auf Finish (Fertigstellen).
    11. Bestätigen Sie, dass die Anwendung ist Aktiv und aktiviert.

    Die Zuordnung der Anwendungsattribute führt dazu, dass in Brightcove Engage Experiences Folgendes geschieht:

    • Bei Mobile App Experiences werden der Name und die E-Mail-Adresse des Benutzers auf der Profilseite angezeigt
    • Für alle Erlebnisse wird die E-Mail-Adresse des Benutzers als Teil der Videoanalyse angezeigt

    Weisen Sie das Access Control-Profil einem Erlebnis zu

    Der letzte Schritt besteht darin, das erstellte Zugriffssteuerungsprofil den Erlebnissen zuzuweisen, die Sie sichern möchten.

    1. Zurück zur Seite Brightcove Engage
    2. Bearbeiten Sie das Erlebnis, das Sie sichern möchten.
    3. Klicken Sie im linken Navigationsbereich auf SITE CONFIGURATION > Access Control.
    4. Wählen Sie das Access Control Profile aus, das mit der PingOne SSO-Konfiguration verknüpft ist.
    5. Klicken Sie auf Save.
    6. Veröffentlichen Sie die Erfahrung.

    Wenn Sie auf das Erlebnis zugreifen, sollten Sie von PingOne aufgefordert werden, sich anzumelden. Das folgende Bild zeigt sich beim Versuch, auf eine Mobile App Experience zuzugreifen.
