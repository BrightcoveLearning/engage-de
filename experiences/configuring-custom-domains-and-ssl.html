Benutzerdefinierte Domänen und SSL konfigurieren

In diesem Thema erfahren Sie, wie Sie benutzerdefinierte Domänen und SSL für ein Web-Erlebnis konfigurieren.

Wenn Sie zum ersten Mal eine Web-Erfahrung erstellen, wird ihr eine zufällige Domäne mit dem Format von zugewiesen <site-slug>.bcvp0rtal.com . Zum Beispiel http://site-148393.bcvp0rtal.com. Wenn das Erlebnis veröffentlicht wird, können die Zuschauer auf die Website dieser Domain zugreifen. Brightcove Engage bietet die Möglichkeit, Web-Erlebnisse in einer von Brightcove bereitgestellten Domain oder in Ihrer eigenen benutzerdefinierten Domain zu veröffentlichen.

Web-Erlebnisse können auch mehreren URLs zugewiesen werden, die es Publishern ermöglichen, dieselbe Site für verschiedene Funktionen zu verwenden, ohne die Website duplizieren zu müssen. Wenn Sie beispielsweise eine Videoseite haben, die Sie in einer Kampagne und als Hauptvideoportal verwenden möchten, können Sie verschiedene URLs markieren und sie alle derselben Website zuweisen. Es ist keine Site-Duplizierung erforderlich und Sie können URLs einfach löschen, wenn sie nicht mehr benötigt werden.

Verwenden einer von Brightcove bereitgestellten Domäne

Mit Brightcove bereitgestellten Domains können Sie schnell und einfach eine Domain für Ihr Web-Erlebnis erstellen, ohne mit Ihrer IT-Abteilung zusammenarbeiten zu müssen, um einen benutzerdefinierten CNAME zu erstellen oder einen neuen Domainnamen bei einem Registrar zu registrieren. Sie sind „out of the box“ verfügbar und können mit einem Klick auf eine Schaltfläche implementiert werden. Die verfügbaren Domains sind:

  • brightcoveengage.video
  • bcvp0rtal.com (das standardmäßig Web-Erlebnisse zugewiesen ist)

Alle von ihnen unterstützen SSL. Um auf die SSL-Site zuzugreifen, fügen Sie einfach https zur URL hinzu. Zum Beispiel https://mysite.brightcoveengage.video

Verwenden einer benutzerdefinierten Domäne (Herausgeber registriert)

Web Experiences können auch auf Ihrer eigenen benutzerdefinierten Domain gehostet werden, die Sie registrieren. Auf diese Weise können Sie die URL brandmarken und sie mit Ihrem aktuellen Website/URL-Namenskonstrukt konsistent halten. Dies erfordert, dass Sie den Domainnamen registrieren und dann einen benutzerdefinierten CNAME-Datensatz bei Ihrem Hosting-Anbieter konfigurieren. Im folgenden Abschnitt finden Sie Informationen zur Konfiguration eines CNAME-Datensatzes.

Erstellen einer benutzerdefinierten Domäne

Gehen Sie folgendermaßen vor, um eine benutzerdefinierte Domäne zu erstellen:

  1. Öffne Brightcove Engage.
  2. Öffnen Sie die Seite Projekte und klicken Sie oben auf der Seite auf den Link Einstellungen
  3. Klicken Sie in der linken Navigation auf den Link Benutzerdefinierte Domains und SSL.
  4. Klicken Sie auf Benutzerdefinierte Domäne erstellen.
  5. Wählen Sie eine zu verwendende Domänenoption aus:
    • brightcoveengage.video - Die Site-Domain wird das Format haben <your domain>.brightcoveengage.video.
    • Benutzerdefiniert - Wählen Sie aus, um die Website auf Ihrer eigenen benutzerdefinierten Domain zu hosten. Wenn Sie diese Option auswählen, müssen Sie auch einen CNAME-Datensatz bei Ihrem Hosting-Anbieter konfigurieren.
  6. Geben Sie den Domainnamen ein. Es wird eine Meldung angezeigt, die angibt, ob die Domäne gültig oder ungültig ist. Wenn der Domainname ungültig ist, geben Sie einen neuen Domainnamen ein.
  7. Wenn Sie die Website mit einer von Brightcove bereitgestellten Domäne veröffentlichen, besteht die Möglichkeit, SSL für diese Domäne durchzusetzen. Aktivieren Sie SSL in dieser Domäne erzwingen , um eine Weiterleitung von der Nicht-SSL-Version der Site zu aktivieren, wenn ein Viewer zu dieser Website geht.
  8. Wenn Sie die Website in Ihrer eigenen benutzerdefinierten Domain veröffentlichen, haben Sie die Möglichkeit, die Website mit SSL zu sichern. Aktivieren Sie das Kontrollkästchen SSL verwenden, um die Website mit SSL zu sichern. Informationen zur Verwendung von SSL finden Sie unter Sichern einer Weberfahrung mit SSL.
  9. Aktivieren Sie Weiterleitung aktivieren, um diese URL an eine andere URL umzuleiten. In der Regel würde dies erst überprüft, nachdem diese Domain eine Weile verwendet wurde. Wenn diese Option ausgewählt ist, geben Sie die Umleitungs-URL ein, zu der diese vorhandene URL umgeleitet werden soll. Aktivieren Sie Eine permanente Weiterleitung verwenden (HTTP-Statuscode 301), um anzugeben, dass die Weiterleitung dauerhaft ist. Wenn eine Suchmaschinenspinne einen 301-Statuscode im Antwortheader einer Webseite findet, versteht sie, dass diese Webseite nicht mehr existiert und ersetzt die indizierte URL durch die neue. Diese Option behält Ihren alten Webseitenverkehr und Ihren Seitenrang bei.
  10. Klicken Sie auf Speichern. Eine Liste der konfigurierten Domänen wird angezeigt.
    Hinweis: Wenn Sie eine von Brightcove bereitgestellte Domäne verwenden, wird der SSL-Status immer bereitgestellt. Dies zeigt an, dass durch das Hinzufügen von https:// zum Anfang der Site-URL die Datenübertragung zwischen Ihrem Browser und der Brightcove Engage-Website sicher ist.
  11. Erstellen Sie nach Bedarf zusätzliche Domains.

Um eine Domäne zu bearbeiten oder zu löschen, klicken Sie auf die Bearbeitung () oder löschen () Knöpfe.

Sobald benutzerdefinierte Domänen erstellt wurden, können sie als Teil der Experience-Eigenschaften einem Erlebnis zugewiesen werden. Weitere Informationen finden Sie unter Zuweisen einer benutzerdefinierten Domäne zu einer Weberfahrung.

Konfigurieren von CNAME-Datensätzen mit Ihrem Hosting-Anbieter

Hinweis: CNAME-Datensätze müssen nur konfiguriert werden, wenn benutzerdefinierte Domänen einer Site zugewiesen werden. CNAME-Datensätze müssen NICHT für Websites erstellt werden, die von Brightcove bereitgestellte Domains verwenden.

Die Schritte zum Konfigurieren eines CNAME-Datensatzes (Canonical Name) unterscheiden sich je nach Ihrem Hosting-Anbieter. Sie müssen für jede benutzerdefinierte Domäne, die Sie verwenden, einen CNAME-Datensatz konfigurieren. Wenn Ihre Site für die Verwendung von SSL konfiguriert ist, verweist der CNAME-Datensatz auf eine CloudFront-URL. Weitere Informationen finden Sie unter Sichern einer Web-Erfahrung mit SSL. Beachten Sie, dass benutzerdefinierte Domänen eine Subdomain enthalten müssen, um als CNAME konfiguriert zu werden, ex: engage muss vorher platziert werden, brightcovelearning.com damit ich meine CNAME.

Der CNAME-Wert wird im Rahmen der Einstellungen für benutzerdefinierte Domänen und SSL angezeigt.

In diesem Beispiel würde ich für die benutzerdefinierte Domäne von engage.brightcovelearning.com einen CNAME-Datensatz hinzufügen und eine Vielzahl von engage und weisen Sie darauf hin 5e84f4c7d58950766328a37d.bcvp0rtal.com.

Möglicherweise müssen Sie sich an Ihren Netzwerkadministrator oder Webhosting-Anbieter wenden, um Hilfe bei diesem Schritt zu erhalten. Denken Sie daran, dass es bis zu 48 Stunden dauern kann, bis DNS-Änderungen wirksam werden.

Sichern von Websites mit SSL

Hinweis: Wenn Sie mit SSL-Zertifikaten arbeiten, teilen Sie Ihren privaten Schlüssel unter keinen Umständen oder senden Sie ihn an den Brightcove Support.

Es gibt zwei Möglichkeiten, ein Portal Experience mit SSL zu sichern:

  1. Für Websites, die mit dem zugewiesenen Site-Slug (d. h. <site-slug>.bcvp0rtal.com) veröffentlicht werden oder eine von Brightcove bereitgestellte Domäne (<site>.brightcoveengage.video) verwenden, ist keine Konfiguration benötigt, um https zu unterstützen. Das Hinzufügen von https:// am Anfang der URL sichert die Datenübertragung zwischen Ihrem Browser und dem Erlebnis. Sie können auch SSL in dieser Domäne erzwingen (empfohlen) auswählen, um eine Weiterleitung von der Nicht-SSL-Version der Site zu aktivieren, wenn ein Viewer zu ihm geht.
  2. Aktivieren Sie für Websites, die in einer benutzerdefinierten Domäne gehostet werden, die Option SSL verwenden, wenn Sie Ihren Domainnamen eingeben. Sie werden dann aufgefordert, Ihre Zertifikatsinformationen einzugeben. Weitere Informationen finden Sie unter Sichern einer Web-Erfahrung mit SSL.
