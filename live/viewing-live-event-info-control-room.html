Anzeigen von Live-Ereignisinformationen im Kontrollraum

In diesem Thema erfahren Sie, wie Sie Live-Ereignisinformationen im Kontrollraum anzeigen.

Die Seite „Control Room“ wird verwendet, um Informationen über ein Live-Ereignis anzuzeigen. Wenn Sie ein neues Live-Event erstellen und das Streaming beginnt, wird die Seite Control Room geöffnet, in der die Ereignisinformationen angezeigt werden. Eine Vorschau des Ereignisses wird im Vorschau-Player angezeigt. Die Seite Kontrollraum wird auch angezeigt, wenn Sie auf der Seite Live-Ereignisse verwalten auf ein Ereignis klicken.

Die Kontrollraumseite Informationen zur Veranstaltung Die Registerkarte ist in folgende Abschnitte unterteilt:

  • Informationen zur Veranstaltung
  • Informationen streamen
  • Clips
  • Monitor

Anzeigen von Ereignisinformationen

Das Informationen zur Veranstaltung Abschnitt zeigt die folgenden Informationen:

  • Veranstaltungsname - Name der Veranstaltung
  • kurze Beschreibung - Kurze Beschreibung der Veranstaltung
  • Stichworte - Tags, die dem Ereignis zugeordnet werden sollen. Tags erleichtern die Suche und können beim Erstellen intelligenter Wiedergabelisten verwendet werden.
  • Referenz ID - Interne Referenz-ID für das Ereignis
  • Streaming zu Social Media - Wenn Dürfen kann das Live-Event einem geplanten sozialen Event zugeordnet werden
  • Digital Rights Management (DRM) - -
  • Wiedergabe-URL - URL, über die der Live-Stream in einem Nicht-Brightcove-Player angezeigt werden kann

Anzeigen von Stream-Informationen

Das Informationen streamen Abschnitt zeigt die folgenden Informationen:

  • Job-ID - Interne Job-ID
  • Video ID - Video-ID des Live-Event-Streams (dies ist auch die Video-ID für das VOD-Asset auf der Videoseite)
  • Endpoint (RTMP URL) - Streaming endpoint used by the encoder
  • Name - Stream name used by the encoder (always alive)
  • Region - Streaming-Region
  • Status - Aktueller Jobstatus
    • Streaming - Stream ist live
    • Verarbeitungsfehler - Beim Livestream ist ein Fehler aufgetreten
    • Stehen zu - Stream bereit zur Aktivierung (nur statische Einstiegspunkte)
    • Anhalten - Der Live-Stream wird gestoppt und erstellt alle zugehörigen VOD-Ausgaben
    • Vollendet - Livestream ist abgeschlossen
    • Stornieren - Der Livestream wird gestoppt und verarbeitet KEINE ausstehenden VOD-Ausgaben
    • Abgesagt - Der Livestream wurde abgebrochen
    • Getrennt - Der Encoder wurde getrennt
    • Warten - Warten auf den RTMP-Stream (Encoder startet)
    • Gescheitert - Der Job wurde aufgrund eines Systemfehlers gestoppt

Clips anzeigen

Das Clips Im Abschnitt werden alle Clips angezeigt, die (falls vorhanden) aus dem Live-Stream erstellt wurden. Durch Klicken auf einen Clipnamen wird der Clip auf dem geöffnet Videos Seite. Informationen zum Erstellen eines Clips finden Sie unter Erstellen von Videoclips.

Überwachung des Streams

Das Monitor Im Abschnitt werden die Diagramme "Ausgabe" und "Eingabedrift" angezeigt. Drücke den Rufen Sie aktualisierte Daten und Grafiken ab Link, um die Informationen anzuzeigen. Output Drift misst die Gesundheit des Streams. Die Ausgangsdrift ist definiert als die Differenz zwischen der Wanduhr und den Stream-Zeitstempeln (TS). Beim Live-Streaming sollte die Wanduhr idealerweise im gleichen Tempo wie der Stream TS laufen. Wenn der Stream TS langsamer als in Echtzeit ankommt, bedeutet dies, dass ein Problem (Engpass) im System vorliegt.

Die Eingangsdrift misst den Zustand des Eingangsstroms und ist definiert als die Differenz zwischen dem Eingangszeitstempel und der Echtzeituhr.
